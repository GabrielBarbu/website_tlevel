# Features Overview and Requirements Mapping

**Project:** Health Advice Group - Digital Solution Prototype
**Date:** 2025-10-08

This document maps the client's requirements from the project brief to the specific features implemented in the final functional prototype.

---

### Client's Core Needs

The client requested a digital solution that provides the following:

| Requirement                               | Implemented Feature                                                                                                                            | File(s)                                                              | Status      |
| ----------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------- | ----------- |
| Weather forecasting                       | A dedicated page where users can enter a location and receive a real-time weather forecast, including temperature, condition, and humidity.        | `weather.php`                                                        | **Completed** |
| Access to a dashboard for air quality     | A dedicated page for monitoring air quality, which is also featured as a main card on the central user dashboard for easy access.                 | `air-quality.php`, `index.php`                                       | **Completed** |
| Advice on health matters                  | A dedicated page that provides personalized health advice by combining real-time weather and air quality data and querying a database for relevant tips. | `health-advice.php`                                                  | **Completed** |

---

### Market Research Features

The client's market research identified the following potential features:

| Requirement                               | Implemented Feature                                                                                                                                                                                                                         | File(s)                                                              | Status                |
| ----------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------- | --------------------- |
| Personalised health advice based on location | The Weather, Air Quality, and Health Advice pages were all updated to accept geographic coordinates, allowing for precise, location-aware data. A "Use My Location" feature was initially implemented but later removed due to environment restrictions. | `weather.php`, `air-quality.php`, `health-advice.php`                | **Completed**         |
| A personal health tracking tool           | A secure, user-specific health tracking system was implemented. Logged-in users can add, view, and manage their own private health logs, which are protected by Supabase's Row-Level Security.                                                 | `health-tracker.php`, `add_health_entry.php`                         | **Completed**         |
| Accessibility features                    | The UI was built with a clean, modern design using semantic HTML and a clear visual hierarchy. The color scheme provides good contrast, and form elements are correctly labelled. This provides a strong baseline of accessibility.           | `css/style.css`, all `.php` files                                    | **Partially Completed** |
